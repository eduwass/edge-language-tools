**edge-language-tools** brings type safety and editor tooling to [EdgeJS](https://edgejs.dev) — red squiggles in your editor, a CI checker, and typed `edge.render()` calls. Works with bare `edge.js`; no AdonisJS required.

Declare a template's interface once, in a comment today's Edge already ignores:

```edge
{{--
@types {
  user: import('#models/user').User
  items: string[]
}
--}}

<h1>{{ user.name }}</h1>
@each(item in items)
  <li>{{ item }}</li>
@end
```

Everything below the block is inferred by TypeScript: `@let` locals, `@each` item types, narrowing in `@if`. A typo like `user.nmae` gets a diagnostic on the exact characters — in the editor and in CI.

<Panel title="Gradual by design">
  Templates without a `@types` block stay unchecked. Adoption is template-by-template and can never break rendering — the block is tooling-only metadata, invisible to the Edge runtime.
</Panel>

## How it works, in one sentence

Each template compiles to a virtual TypeScript module — expressions copied verbatim with exact offset mappings, constructs emitted as real control flow — so TypeScript does the checking and [Volar.js](https://volarjs.dev) serves it to editors. Read [How it works](/how-it-works) for the details.
